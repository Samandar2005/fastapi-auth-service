# Contributing

Thanks for your interest in contributing to this project! A few simple guidelines:

1. Fork the repository and create a new branch for your change (feature/ or fix/).
2. Write tests for new features or bug fixes when possible.
3. Run linters and tests locally before opening a PR.
4. Follow existing code style and keep changes minimal.
5. Open a Pull Request describing your change and why it's needed.

Maintainers will review PRs and provide feedback. Thank you!
